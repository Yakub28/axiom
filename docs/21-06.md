## 2. What was built (phase by phase)

### Phase 0 — Config + dependencies
- `axiom/config.py`: added an OpenAlex block — `OPENALEX_BASE`, `OPENALEX_MAILTO`
  (polite pool, no API key needed), `SEED_QUERY`, `SEED_COUNT`, `CORPUS_TARGET=500`,
  `CITERS_PER_SEED`, `OPENALEX_BATCH_SIZE`, timeouts/retries.
- `requirements.txt`: added `httpx==0.27.0` (HTTP) and `networkx==3.6.1` (graph).
  `pyvis` was added then **removed** (see §5, viz pivot to 3D).

### Phase 1 — OpenAlex ingestion (`axiom/ingest.py`)
Pure-Python (no torch), so the fetch path runs on any Python version.
- **HTTP client** with polite-pool `mailto`, cursor paging, retry/back-off on
  429/transient errors, and a `select=` field mask to keep responses small.
- **Abstract reconstruction** from OpenAlex's `abstract_inverted_index`
  (`{word: [positions]}` → position-ordered plain text).
- **`parse_work`** normalizes a raw work to our schema columns (id short-form,
  venue from `primary_location.source`, concepts, `referenced_works`).
- **Snowball / 2-hop expansion** (`snowball`): seed on a topic, then per hop pull
  (a) capped citers of frontier papers and (b) cited works ranked by
  **co-citation frequency**, until ~`CORPUS_TARGET` papers. This is what makes
  the in-corpus graph *connected* rather than a star of dangling edges.
- **`ingest_to_sqlite`** writes through the frozen `db.insert_*` helpers
  (idempotent wipe + reload). Citation edges keep **all** `referenced_works`,
  including dangling externals (no FK on `dst_id` by design).

### Phase 1b — Shared indexer (`axiom/indexer.py`)
- Extracted the embed + sparse-BM25 + Qdrant-upsert logic into
  `reindex_qdrant()`, used by **both** the synthetic bootstrap and the new
  ingestion CLI (removed ~50 lines of duplication from `bootstrap_synthetic.py`).
- **Lazy** torch/transformers imports: importing the module is cheap; only
  calling `reindex_qdrant()` needs the ML stack (Python 3.10–3.12).

### Phase 1c — Ingestion CLI (`scripts/ingest_openalex.py`)
- `--topic`, `--target`, `--seeds`, `--no-index`.
- Fetch → SQLite always runs; embedding step degrades gracefully with a clear
  message if torch is unavailable (e.g. the 3.14 dev env).

### Phase 2 — Citation graph (`axiom/graph.py`)
Pure NetworkX (no torch, importable anywhere).
- `load_graph()` → `networkx.DiGraph`, edge `src→dst` = "src cites dst";
  in-corpus nodes carry metadata + `in_corpus=True`, dangling externals are bare
  nodes flagged `in_corpus=False`.
- `corpus_subgraph()` — induced subgraph of in-corpus nodes only.
- `influence()` — PageRank over the corpus-only subgraph + `local_in_degree`.
- `neighbors()` — references (successors) vs cited-by (predecessors).
- `top_influential_subgraph()` / `ego_subgraph()` — small drawable subgraphs.
- `stats()` — health numbers.

### Phase 3 — Influence ranking in Streamlit
- New **"🕸️ Citation graph"** tab (search moved under a "🔍 Search" tab).
- Graph stat metrics + PageRank-ranked papers; each expander shows DOI, local vs
  global citation counts, and the top corpus papers citing it.
- Cached via `@st.cache_resource` (graph) and `@st.cache_data` (ranking).

### Phase 4 — Interactive 3D visualization
- Per-paper **3D force-directed graph** via `3d-force-graph` (WebGL/three.js,
  loaded from CDN) embedded with `st.components.v1.html`.
- Two views: **Top influential** (top-N PageRank subgraph) and **Around a paper**
  (1-hop ego network). Node size = in-degree within the view; gold particles flow
  citing → cited; click a node to fly the camera to it.

## 4. Bugs caught and fixed

1. **Off-topic seeds.** Sorting seeds by `cited_by_count:desc` returned the
   most-cited papers in *all of OpenAlex* that merely contained the query tokens
   (SciPy, QUANTUM ESPRESSO, "Data clustering"). Fixed by seeding with the
   `title_and_abstract.search` filter using OpenAlex's relevance ranking → on-topic
   RAG papers.
2. **PageRank pulled in `scipy`** (not installed; breaks "pinned, minimal" deps).
   Replaced `nx.pagerank` with a small **pure-Python power iteration**.
3. **PageRank direction bug.** Initially scored the *reversed* graph; standard
   PageRank already accumulates rank on nodes with many incoming edges (= cited
   papers), so it must run on the original direction. Fixed.
4. **3D graph rendered black.** Two causes: (a) no explicit canvas height inside
   the Streamlit iframe; (b) `three-spritetext` labels expected a global `THREE`
   the bundle doesn't expose, throwing per node. Fixed by setting explicit
   width/height, removing spritetext, and adding a visible load-failure fallback.

## 11. Course-correction: research-gap detection (the actual key value)

Reviewed the day's work against the project goal ("extract keywords/topics/
datasets; help identify **research gaps**"). Honest assessment: ingestion + the
graph + 3D viz are correct *foundation*, but influence ranking finds the most
*central* papers — the opposite of a gap — and nothing yet delivered the key
value. Course-corrected by pointing the existing pieces at gaps:

- **`axiom/gaps.py`** (new) — Louvain communities on the citation graph,
  TF-IDF-distinctive concept labels, embedding centroids, and a pairwise
  `gap_score = cosine / (1 + inter_citations)`. See decision **OD9**.
- **`AxiomQdrant.fetch_dense_vectors()`** (new) — pulls stored embeddings for
  community centroids.
- **UI** — graph tab now leads with a **"Research gaps"** view: nodes coloured by
  sub-topic cluster, a candidate-gap picker that renders the two related-but-
  disconnected clusters in 3D, a "why this is a gap" explainer with the top
  papers on each side, and a ranked list of all candidates.

First real run (500-paper RAG corpus): **12 communities** with interpretable
labels (medical/MEDLINE, knowledge-base/NER/Transformer, aviation maintenance,
telecom, mental health, education) and plausible candidates — e.g.
*aviation-maintenance RAG ⟷ telecom RAG*, semantically close, **0 citations
between them**. Honest caveat (OD9): on a sparse single-topic corpus the semantic
axis barely varies (~0.95) and most pairs have 0 inter-citations, so the signal
is weak; denser/multi-topic corpora will sharpen it.

**Still not addressed:** dataset/keyword *extraction* (the other half of the goal)
is a separate pipeline, not started.

## 12. Identified next: end-to-end topic-driven workflow

Reviewing the tool against the goal surfaced a **product gap, not a code gap**:
the analytical pieces all exist, but the topic is chosen at *ingestion* time via
a CLI flag (`--topic`) — the Streamlit app only analyzes whatever corpus is
already loaded. There is no in-app "enter a field → go" front door, so a user
can't get from *"I care about underrepresented languages"* to results without
dropping to a terminal.

Desired workflow (the user inputs the field and the tool drives the rest):

```
1. INPUT      type a field/topic            ← e.g. "underrepresented languages"
2. BUILD      snowball OpenAlex → corpus     ✅ axiom/ingest.py    (CLI-only today)
3. EMBED      SPECTER2 → Qdrant              ✅ axiom/indexer.py   (CLI-only today)
4. MAP        citation graph + communities   ✅ axiom/graph.py, gaps.py
5. FIND GAPS  related-but-disconnected        ✅ axiom/gaps.py
6. EXPLORE    search / influence / 3D / gaps  ✅ app/streamlit_app.py
```

Everything in 4–6 is built; the missing piece is the **front door wiring 2–3
into the UI** and refreshing 4–6 for the chosen field.

Open design decisions (to settle before building):
- **Where the build runs:** in-app on-demand (topic input + "Analyze field"
  button, ~2–3 min with a progress bar; needs torch + Qdrant) vs. keep ingestion
  a CLI prep step and have the app just show/select the active corpus.
- **Corpus model:** one active field that a new topic *replaces* (matches the
  current idempotent design, ships fastest) vs. multiple fields side by side
  (collection-per-topic, more management).
- **On-demand corpus size:** smaller (≈200) builds faster for interactive use;
  larger (≈500+) gives a denser graph but a slower build.

Status: **not yet built** — design decision pending.

# Supervisor Bi-Weekly Update — Presentation Input

> Copy-ready talking points / slide bullets. Audience: project supervisor.

### Slide 1 — What we set out to do
- Add a **citation graph** to Axiom — and do it on **real literature**, not the
  synthetic placeholder corpus, so the analysis actually means something.

### Slide 2 — What we delivered (this sprint)
- **Real corpus**: ingested **~500 papers (28k citations)** on retrieval-augmented
  generation directly from **OpenAlex**, replacing the 30 synthetic papers.
- **Citation graph engine**: in-memory graph (NetworkX) with **PageRank-based
  influence ranking** — surfaces the *structurally* foundational papers.
- **First research-gap detector**: clusters the field into sub-topics and flags
  pairs that are **related in meaning but don't cite each other** — candidate
  openings for a thesis (the project's key value).
- **Interactive 3D visualization**: rotate/zoom/click a live citation graph,
  coloured by sub-topic, with a "research gaps" view that shows the disconnected
  clusters directly.

### Slide 3 — Key result / the "so what"
- PageRank ≠ raw citation counts. Example: **"Stochastic RAG" ranks #3 in
  influence despite only 6 in-corpus citations**, because it's cited by *central*
  papers. Raw counts miss this; the graph surfaces hidden hubs.
- This is the **foundation for gap detection** (Axiom's end goal): you can't find
  the "holes" in a field until you can map its structure.

### Slide 4 — Engineering decisions worth noting
- **Snowball ingestion** (seed → cited + citing, 2 hops): the only way to get a
  *connected* graph — a naive topic fetch produces edges that all point outside
  the set and tells you nothing.
- **Kept the data contract frozen**: no schema changes; new code plugs into the
  existing seams. Two decisions logged (OD7, OD8) for the team.
- **Dependency discipline**: implemented PageRank by hand rather than pull in a
  heavy library; kept the pinned, minimal stack.

### Slide 5 — Challenges & lessons
- **Relevance trap**: sorting search seeds by citation count returned famous-but-
  off-topic papers (SciPy, etc.). Switched to relevance-ranked seeding.
- **Visualization is iterative**: first 2D attempt was unreadable; moved to 3D,
  then did a real accessibility/contrast pass (edge visibility, legends, controls).
- Real data is ~84% abstract coverage and sparse cross-citation (4.4% in-corpus) —
  expected for a recent, fast-moving topic; informs corpus sizing going forward.

### Slide 6 — Next steps
- **End-to-end "enter a field → analyze" workflow** — wire the existing pipeline
  (fetch → embed → graph → gaps) behind a single topic input in the app, so a
  researcher goes from *"underrepresented languages"* to gaps without a terminal.
  This is the front door that ties everything we built into one usable tool.
- **Dataset & keyword extraction** — the other half of the goal; not yet started
  (a separate NLP pipeline over abstracts/full text).
- **Sharpen gap detection** — denser/multi-topic corpora + better semantic
  discrimination so candidate gaps are sharper and more defensible.
- **Citation velocity** — use the citation years we already store to rank
  *trending* (vs. dormant) gaps.
- **Validate gaps with a domain expert** — confirm a flagged gap is genuinely
  under-explored, not just a corpus artifact.

### One-line summary
> "We moved Axiom onto real data and gave it a citation graph: it can now rank the
> truly influential papers in a field and show that structure in interactive 3D —
> the groundwork for finding research gaps."
